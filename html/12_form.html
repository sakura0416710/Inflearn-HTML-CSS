method : 데이터를 전송할 방법. get과 post 두가지 방법
1. get : 전송하는 데이터가 url에 붙어서 전송되므로 데이터가 url에 노출된다. 보안에 취약하고 페이지 주소를 공유해야할 때 보안에 취약함.
페이지 주소를 공유해야할 때 주로 사용한다. 전송가능한 데이터 크기에 제약있음
2. post : 전송하는 데이터가 url에 붙지 않으므로 사용자 눈에 보이지 않는다. 따라서 보안적으로 상대적으로 안정적이고(개발자 도구에선 보임;)
결국은 전송하는 데이터를 확인가능하므로 전송하는 데이터는 암호화하는 것이 좋다. 전송가능한 데이터 크기에 제약없음
*https프로토콜은 암호화해서 데이터를 전송해줌
